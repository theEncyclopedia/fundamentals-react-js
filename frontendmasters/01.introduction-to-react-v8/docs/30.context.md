# React Context: Global State Management

React Context is a tool for managing global state in a React application. It simplifies the process of sharing state between deeply nested components without needing to pass props explicitly through multiple layers of components. This document demonstrates how to use React Context by creating a context for managing the adoption status of pets across the app.

## Key Topics

### 1. What Is Context?

- **Definition:**

  - A mechanism to share state globally in a React app.
  - Avoids "prop drilling" by providing a centralized source for state.

- **When to Use Context:**
  - State required across multiple components (e.g., user authentication, themes).
  - Avoid overusing it for local component state.

### 2. Creating Context

#### Step 1: Set Up Context

- Create a new file `AdoptedPetContext.js`.
- Use `createContext` to define a context.

  ```jsx
  import { createContext } from 'react';

  const AdoptedPetContext = createContext();

  export default AdoptedPetContext;
  ```

#### Step 2: Provide Context in the App

- Wrap the app’s components with a context provider in `App.jsx`.
- Use `useState` to manage the value passed to the context.

  ```jsx
  import React, { useState } from 'react';
  import AdoptedPetContext from './AdoptedPetContext';

  function App() {
    const adoptedPet = useState(null);

    return (
      <AdoptedPetContext.Provider value={adoptedPet}>{/* Rest of your app components */}</AdoptedPetContext.Provider>
    );
  }

  export default App;
  ```

### 3. Consuming Context

#### Using Context in Components

- Use `useContext` to access the context value.
- Example in `Details.jsx`:

  ```jsx
  import React, { useContext } from 'react';
  import AdoptedPetContext from './AdoptedPetContext';

  function Details() {
    const [adoptedPet, setAdoptedPet] = useContext(AdoptedPetContext);

    return (
      <div>
        <h1>Would you like to adopt this pet?</h1>
        <button onClick={() => setAdoptedPet('Luna')}>Adopt Luna</button>
      </div>
    );
  }

  export default Details;
  ```

### 4. Managing Context State

- **What Can Context Hold?**

  - Any JavaScript value: objects, arrays, strings, numbers, or even hooks.

- **Best Practices:**
  - Use context for truly global state.
  - Avoid using it for state only relevant to a single component.

### 5. Benefits and Trade-offs

#### Benefits

1. **Simplifies Prop Drilling:**
   - Eliminates the need to pass props through intermediate components.
2. **Centralized State:**
   - Provides a single source of truth for global state.

#### Trade-offs

1. **Indirection:**
   - Makes it harder to trace data flow.
2. **Performance:**
   - Frequent updates to large contexts can cause unnecessary re-renders.

## Lessons Learned

1. **When to Use Context:**
   - For app-wide state like user authentication, themes, or global settings.
2. **Avoid Overuse:**
   - For state that only affects a small part of the app, use `useState` or `useReducer` locally.
3. **Pair with Hooks:**
   - Combine `useState` with context to manage state seamlessly.

## Summary

React Context is a powerful tool for managing global state, but it should be used judiciously to avoid unnecessary complexity. By understanding its strengths and limitations, developers can create scalable and maintainable React applications.
