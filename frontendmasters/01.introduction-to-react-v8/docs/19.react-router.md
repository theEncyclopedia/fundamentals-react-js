# Routing in React with React Router

This section explains how to implement routing in React using React Router, covering setup, usage, and best practices.

## Why Use React Router?

React Router is the most popular library for managing routes in single-page applications. It allows:

- Navigating between different views (pages) without full-page reloads.
- URL-based routing with dynamic parameters.
- A declarative and flexible approach to routing.

## Installing React Router

To install React Router DOM version 6:

```sh
npm install react-router-dom@6.4.1
```

## Basic Setup

### Wrapping the App with `BrowserRouter`

In `App.jsx`, wrap the application with `BrowserRouter` to enable routing:

```jsx
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import SearchParams from './SearchParams';
import Details from './Details';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<SearchParams />} />
        <Route path="/details/:id" element={<Details />} />
      </Routes>
    </BrowserRouter>
  );
}

export default App;
```

### Explanation

1. **`BrowserRouter`**:

   - Provides the routing context for your app.
   - Wraps all components where routing is needed.

2. **`Routes`**:

   - Acts as a container for `Route` components.

3. **`Route`**:

   - Defines individual routes with a `path` and a component (`element`) to render.

4. **Dynamic Parameters**:
   - The `:id` in `/details/:id` is a dynamic parameter accessible in the `Details` component.

## Using Links Instead of `<a>` Tags

React Router provides the `Link` component to enable client-side navigation without full-page reloads. Replace `<a>` tags with `Link` for better performance.

### Example: Refactoring the `Pet` Component

Replace `<a>` with `<Link>`:

```jsx
import { Link } from 'react-router-dom';

function Pet({ id, name, animal, breed, images, location }) {
  const hero = images.length ? images[0] : 'http://pets-images.dev-apis.com/pets/none.jpg';

  return (
    <Link to={`/details/${id}`} className="pet">
      <div className="image-container">
        <img src={hero} alt={name} />
      </div>
      <div className="info">
        <h1>{name}</h1>
        <h2>{`${animal} — ${breed} — ${location}`}</h2>
      </div>
    </Link>
  );
}

export default Pet;
```

### Benefits of `Link`

- Avoids full-page reloads.
- Maintains React state without reinitializing the app.
- Improves performance on low-end devices.

## React Router Features and Best Practices

### 1. Dynamic Routing

React Router allows dynamic path parameters (e.g., `/details/:id`) to access route-specific data.

### 2. Accessibility

React Router is highly accessibility-focused, ensuring compatibility with screen readers and other assistive technologies.

### 3. Avoiding Full Refresh

Use `Link` instead of `<a>` for client-side navigation to improve performance and user experience.

## React Router Components Overview

1. **`BrowserRouter`**:

   - Standard router for modern browsers.
   - Uses clean URLs without hashes.

2. **`HashRouter`**:

   - Uses URLs with hashes (e.g., `/#/details/123`).
   - Useful for legacy backends or static file hosting.

3. **`Routes` and `Route`**:

   - Define and manage individual routes.
   - Support nested routes and dynamic parameters.

4. **Dynamic Parameters**:
   - Access parameters like `id` in `/details/:id` using hooks like `useParams`.

## Example Workflow

1. Navigate to `/` to display the search page (`SearchParams`).
2. Click a pet to navigate to `/details/:id` using a `Link`.
3. The `Details` component renders with the appropriate `id`.

## Conclusion

React Router is a powerful and flexible library for managing routing in React applications. By leveraging components like `BrowserRouter`, `Routes`, `Route`, and `Link`, you can create seamless, performant, and accessible single-page applications. With a focus on declarative routing and dynamic parameters, React Router is essential for modern React development.
