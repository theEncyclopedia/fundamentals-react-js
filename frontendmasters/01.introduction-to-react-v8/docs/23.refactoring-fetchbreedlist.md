# Refactoring Fetch Logic with React Query

This document explains how to refactor the `fetchBreedList` functionality using React Query for a simpler and more maintainable implementation. The process demonstrates significant improvements in readability and efficiency by replacing manual logic with React Query.

## Refactoring `fetchBreedList`

### Original Function

The original implementation included manual caching, effects, and state management, resulting in a verbose and complex setup.

### Simplified Implementation with React Query

Refactored `fetchBreedList` using React Query:

#### Fetch Function

Create `FetchBreedList.js` to handle fetching breeds from the API:

```jsx
const fetchBreedList = async ({ queryKey }) => {
  const animal = queryKey[1]; // Extract the animal type from the query key
  const apiRes = await fetch(`http://pets-v2.dev-apis.com/breeds?animal=${animal}`);

  if (!apiRes.ok) {
    throw new Error(`breeds/${animal} fetch not ok`);
  }

  return apiRes.json(); // Return the parsed JSON data
};

export default fetchBreedList;
```

### Using `useQuery` in the Component

Update the `useBreedList` custom hook to use React Query:

```jsx
import { useQuery } from '@tanstack/react-query';
import fetchBreedList from './FetchBreedList';

const useBreedList = (animal) => {
  const results = useQuery(
    ['breeds', animal], // Query key
    fetchBreedList, // Fetch function
    {
      enabled: !!animal, // Only fetch if an animal is provided
    },
  );

  return results.data?.breeds ?? []; // Return breeds or an empty array
};

export default useBreedList;
```

### Refactored Component

The `useBreedList` hook simplifies fetching logic in the component:

```jsx
import useBreedList from './useBreedList';

function BreedSelector({ animal }) {
  const breeds = useBreedList(animal);

  return (
    <select disabled={!breeds.length}>
      {breeds.map((breed) => (
        <option key={breed}>{breed}</option>
      ))}
    </select>
  );
}

export default BreedSelector;
```

## Key Concepts

1. **Query Key**:

   - Uniquely identifies the query (`["breeds", animal]`).
   - Enables caching and prevents redundant API requests.

2. **Dynamic Fetch Conditions**:

   - The `enabled` option ensures the query runs only when `animal` is valid.

3. **Optional Chaining and Nullish Coalescing**:
   - `results.data?.breeds ?? []` ensures a fallback to an empty array if data is unavailable.

## Handling Multiple Parameters

To support multiple parameters in the query, extend the query key and fetch logic:

### Example

#### Query Key

```jsx
useQuery(['breeds', animal, region], fetchBreedList);
```

#### Fetch Function

```jsx
const fetchBreedList = async ({ queryKey }) => {
  const [_, animal, region] = queryKey; // Destructure additional parameters
  const apiRes = await fetch(`http://pets-v2.dev-apis.com/breeds?animal=${animal}&region=${region}`);

  if (!apiRes.ok) {
    throw new Error(`breeds/${animal}/${region} fetch not ok`);
  }

  return apiRes.json();
};
```

## Benefits of Refactoring

1. **Simplified Code**:

   - Reduced from 35 lines to under 10 for the same functionality.
   - Eliminated manual caching and effects.

2. **Improved Maintainability**:

   - Centralized logic in the `fetchBreedList` function.
   - Easy to modify for additional parameters or behavior.

3. **Enhanced Performance**:
   - React Query handles caching and avoids redundant API requests.

## Conclusion

Refactoring with React Query makes fetching logic cleaner, easier to maintain, and more scalable. Using features like query keys, conditional fetching, and caching ensures efficient data management while simplifying component code.
