# Using React Query for Data Fetching in React Applications

This section explains how to use React Query to manage API requests and caching in a React application, focusing on setup, configuration, and practical examples.

## What is React Query?

React Query simplifies data fetching and state management by:

- Automatically caching and revalidating API requests.
- Reducing the need for `useEffect` and manual state management.
- Handling background data synchronization and stale data.

## Installing React Query

To install React Query:

```sh
npm install @tanstack/react-query@4.10.1
```

## Setting Up React Query

### Step 1: Import and Instantiate Query Client

In `App.jsx`, import the necessary components and create a query client:

```jsx
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { BrowserRouter } from 'react-router-dom';
import SearchParams from './SearchParams';
import Details from './Details';

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: Infinity, // Never re-fetch automatically
      cacheTime: Infinity, // Keep cached data indefinitely
    },
  },
});

function App() {
  return (
    <QueryClientProvider client={queryClient}>
      <BrowserRouter>
        <Routes>
          <Route path="/" element={<SearchParams />} />
          <Route path="/details/:id" element={<Details />} />
        </Routes>
      </BrowserRouter>
    </QueryClientProvider>
  );
}

export default App;
```

### Explanation

1. **`QueryClient`**:

   - Manages API request caching and state.
   - Accepts configuration options such as `staleTime` (how long data is considered fresh) and `cacheTime` (how long to keep unused data).

2. **`QueryClientProvider`**:

   - Wraps your application to provide React Query context.

3. **`staleTime` and `cacheTime`**:
   - Setting both to `Infinity` means data will not be re-fetched or removed unless explicitly invalidated.

## Fetching Data with React Query

### Example: Creating a Fetch Utility

Create a `FetchPet.js` file for the pet fetching logic:

```jsx
const fetchPet = async ({ queryKey }) => {
  const id = queryKey[1]; // Extract the ID from the query key
  const apiRes = await fetch(`http://pets-v2.dev-apis.com/pets?id=${id}`);

  if (!apiRes.ok) {
    throw new Error(`details/${id} fetch not ok`);
  }

  return apiRes.json();
};

export default fetchPet;
```

### Key Concepts

1. **`queryKey`**:

   - Identifies a unique query (e.g., `["pet", id]`).
   - React Query uses this key for caching and invalidation.

2. **Error Handling**:

   - If the request fails, throw an error to let React Query handle retries or show an error state.

3. **Returning Promises**:
   - Always return a promise from the fetch function. React Query resolves it automatically.

## Using React Query in a Component

Update `Details.jsx` to use React Query for fetching pet details:

```jsx
import { useQuery } from '@tanstack/react-query';
import fetchPet from './FetchPet';

function Details() {
  const { id } = useParams();

  const { data, isLoading, error } = useQuery(['pet', id], fetchPet);

  if (isLoading) return <h2>Loading...</h2>;
  if (error) return <h2>Error fetching pet details</h2>;

  const pet = data?.pets[0];

  return (
    <div>
      <h1>{pet.name}</h1>
      <h2>{`${pet.animal} — ${pet.breed} — ${pet.location}`}</h2>
      <img src={pet.images[0]} alt={pet.name} />
    </div>
  );
}

export default Details;
```

### Explanation

1. **`useQuery`**:

   - Fetches data based on the `queryKey` (e.g., `["pet", id]`) and `fetchPet` function.
   - Provides loading (`isLoading`) and error (`error`) states, along with the fetched data.

2. **Error and Loading States**:

   - Simplifies handling different states without manually managing `useState` and `useEffect`.

3. **Dynamic Query Keys**:
   - The `queryKey` includes dynamic parameters like `id`, ensuring React Query fetches the correct data.

## Benefits of React Query

1. **Automatic Caching**: Prevents redundant API requests and improves performance.
2. **Simplified State Management**: Removes the need for manual `useEffect` and `useState` for data fetching.
3. **Error Handling**: Centralized error handling for API requests.
4. **Flexibility**: Configure caching, refetching, and invalidation behavior to suit your needs.

## Conclusion

React Query significantly improves data fetching and state management in React applications. By leveraging its caching, error handling, and built-in state management features, you can reduce complexity and improve maintainability in your codebase.
