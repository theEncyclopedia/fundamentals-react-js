# Building a React Application Without Build Tools: "Adopt Me"

This section guides you through creating a React application without using build tools, starting with a pure HTML, CSS, and JavaScript setup. We'll build a simple pet adoption app, "Adopt Me," while understanding foundational React concepts and features like `createElement` and reusable components.

## Setting Up the Project

1. **Create the Folder Structure**:

   - Create a folder named `adopt-me`.
   - Inside the folder, create a subdirectory named `src`.
   - Add the following files:
     - `index.html` for the main HTML structure.
     - `style.css` for styling (optional).

2. **Basic HTML Setup**:

   - Use Emmet to generate an HTML5 boilerplate:
     - In VS Code, type `html:5` and press Tab.
   - Replace the title with `Adopt Me`.
   - Add a `<div>` with `id="root"` inside the `<body>`:
     - This acts as the root container for the React application.
   - Link the `style.css` file in the `<head>` section for styling.

3. **Add React and ReactDOM**:

   - Include the following `<script>` tags to load React and ReactDOM:
     - React: Provides the API.
     - ReactDOM: Handles rendering to the DOM.
   - These libraries are globally accessible via `React` and `ReactDOM`.

   Final structure:

   ```html
   <!doctype html>
   <html lang="en">
     <head>
       <meta charset="UTF-8" />
       <meta name="viewport" content="width=device-width, initial-scale=1.0" />
       <title>Adopt Me</title>
       <link rel="stylesheet" href="./style.css" />
     </head>
     <body>
       <div id="root">Not Rendered</div>
       <script src="https://unpkg.com/react@18/umd/react.development.js"></script>
       <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
       <script src="./index.js"></script>
     </body>
   </html>
   ```

## Writing Your First React Component

1. **Define the Component**:

   - Use `React.createElement` to create a functional component named `App`.
   - Always capitalize component names.

   ```jsx
   const App = () => {
     return React.createElement('div', null, React.createElement('h1', null, 'Adopt Me'));
   };
   ```

   Explanation:

   - `React.createElement(type, props, children)`:
     - `type`: The HTML element or component to create (e.g., `div`, `h1`).
     - `props`: Attributes like `id` or `className` (use `null` or `{}` if none).
     - `children`: Content inside the element (e.g., text or other elements).

2. **Render the Component**:

   - Get a reference to the root container and create a root object.
   - Render the `App` component into the root.

   ```jsx
   const container = document.getElementById('root');
   const root = ReactDOM.createRoot(container);
   root.render(React.createElement(App));
   ```

3. **Check the Output**:
   - Open the `index.html` file in a browser.
   - You should see the text "Adopt Me" rendered on the page.

## Understanding React.createElement

- `React.createElement` generates HTML elements programmatically.
- Example:

  ```jsx
  React.createElement('h1', { id: 'title' }, 'Hello World');
  ```

  - Produces:
    - `<h1 id="title">Hello World</h1>`

### JSX vs `createElement`

- JSX is a syntactic sugar for `React.createElement` calls.
- Example:
  - JSX: `<h1>Hello World</h1>`
  - Translates to: `React.createElement("h1", null, "Hello World")`

## Styling the Application

- Add styles in `style.css` to customize the appearance:

  ```css
  body {
    font-family: Arial, sans-serif;
    background-color: #f3f3f3;
  }
  ```

- Link `style.css` in the HTML to apply the styles.

## Advantages of Components

- **Reusable**: Components can be nested and reused to build complex UIs.
- **Scalable**: Small components combine to form larger ones.
- **Maintainable**: Logic is encapsulated within individual components.

## Why Learn `React.createElement`?

- Understanding `React.createElement` demystifies React's rendering process.
- It helps interpret stack traces and debug issues.
- JSX simplifies code but relies on `React.createElement` under the hood.

## Conclusion

This setup introduces core React concepts, including component creation and rendering. Using `React.createElement` builds a foundation for understanding JSX and the broader React ecosystem. You now have a working React application and are ready to explore more advanced features like JSX, state management, and routing.
