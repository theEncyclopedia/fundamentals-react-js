# Managing Dependent Dropdowns in React

This document explains how to manage dependent dropdowns (e.g., for animals and breeds) in React, including enabling, disabling, and resetting the dependent dropdown based on user interaction and state changes.

## Adding a Dependent Dropdown for Breeds

To manage breeds as a dependent dropdown:

1. Create a `breed` state with `useState`.
2. Initialize an empty array for `breeds`, which will later be populated via an API call.

### Example: Adding a Breed Dropdown

```jsx
import { useState } from 'react';

const ANIMALS = ['bird', 'cat', 'dog', 'rabbit', 'reptile'];

function SearchParams() {
  const [animal, setAnimal] = useState('');
  const [breed, setBreed] = useState('');
  const [breeds, setBreeds] = useState([]);

  return (
    <form>
      <label htmlFor="location">
        Location
        <input id="location" />
      </label>

      <label htmlFor="animal">
        Animal
        <select
          id="animal"
          value={animal}
          onChange={(e) => {
            setAnimal(e.target.value);
            setBreed(''); // Clear breed when animal changes
          }}
        >
          <option value="">Select an animal</option>
          {ANIMALS.map((animal) => (
            <option key={animal} value={animal}>
              {animal}
            </option>
          ))}
        </select>
      </label>

      <label htmlFor="breed">
        Breed
        <select
          id="breed"
          value={breed}
          onChange={(e) => setBreed(e.target.value)}
          disabled={breeds.length === 0} // Disable when no breeds are available
        >
          <option value="">Select a breed</option>
          {breeds.map((breed) => (
            <option key={breed} value={breed}>
              {breed}
            </option>
          ))}
        </select>
      </label>
    </form>
  );
}
```

### Key Features

1. **Clearing Dependent State**:

   - The `breed` state is reset to an empty string (`""`) whenever the `animal` state changes.
   - This ensures that the dropdowns remain logically connected (e.g., selecting "Cat" clears previously selected dog breeds).

2. **Disabling the Breed Dropdown**:

   - The `disabled` attribute is set dynamically based on the `breeds` array.
   - If `breeds.length === 0`, the dropdown is grayed out and unavailable for selection.

3. **Dynamic Options**:
   - Both the `animal` and `breed` dropdowns dynamically generate their `<option>` elements using the `map` function.

## Handling Accessibility with `onBlur`

- Previously, the `onBlur` event was recommended to ensure accessibility for users navigating via keyboard (e.g., using the `Tab` key).
- If no errors arise without `onBlur`, it can be omitted for simplicity.

### Example: Adding `onBlur` for Accessibility

```jsx
<select
  id="animal"
  value={animal}
  onChange={(e) => setAnimal(e.target.value)}
  onBlur={(e) => setAnimal(e.target.value)}
>
  ...
</select>
```

## Preparing for API Integration

Currently, the `breeds` array is hardcoded as empty (`const breeds = []`). In a future step, you’ll fetch the list of breeds from an API and update the `breeds` state accordingly.

### Dynamic API-Driven Breeds

When the API call is added, it will:

1. Fetch breeds based on the selected `animal`.
2. Update the `breeds` array with the fetched data.
3. Automatically enable the breed dropdown when data is available.

This setup establishes a strong foundation for creating dynamic and dependent dropdowns in React, preparing you for further integration with external APIs.
