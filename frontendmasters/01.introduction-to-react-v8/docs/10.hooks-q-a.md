# React Hooks and Best Practices

This section will provides an overview of React hooks, their purpose, and best practices for using them effectively in your projects. It also includes tips on setting up ESLint to help ensure proper hook usage and avoid common pitfalls.

## Understanding `const` in React

When you see `const` being used to declare variables in React, note the following:

- `const` in JavaScript does not imply immutability. Instead, it ensures that the reference to the variable remains constant within its scope.
- For example, declaring a variable with `const` in a function means it cannot be reassigned within that function's body. However, it may be reinitialized if the function runs again.

If you're more comfortable with `let`, you can use it without harm—it’s a matter of personal preference.

## What Are Hooks?

Hooks are special functions in React that allow you to manage state and lifecycle features in functional components. They are easily recognizable because their names always start with `use` (e.g., `useState`, `useEffect`).

### Popularity of Hooks

React popularized the concept of hooks, and it has since been adopted by other frameworks like Vue and Svelte. Hooks have become a standard for managing state and logic in modern React development.

## Using `useState`

The `useState` hook allows you to add state to functional components. Here's a breakdown of its usage:

```jsx
const [location, setLocation] = useState(initialValue);
```

This single line destructures the array returned by `useState`:

- `location`: The current state value.
- `setLocation`: The function to update the state.

### Equivalent Explicit Code

```jsx
const locationHook = useState(initialValue);
const location = locationHook[0];
const setLocation = locationHook[1];
```

The shorthand destructuring syntax simplifies this process, but understanding the explicit version can clarify what’s happening.

### Naming Conventions for State Updaters

While naming the state updater function (e.g., `setLocation`), you can use any name that makes sense to you. However, `set` is commonly used for clarity and consistency.

## Creating Custom Hooks

Custom hooks are essentially combinations of other hooks bundled into a single reusable hook. For example:

- Custom hooks can simplify complex state management or logic.
- They are implemented as regular JavaScript functions but must follow the `use` prefix convention.

We'll dive deeper into creating custom hooks later in the course.

## Best Practices for Using Hooks

1. **Avoid Conditional Execution**: Hooks should never be called inside conditionals, loops, or nested functions.
2. **Consistency**: Always start hook names with `use`.
3. **Debugging**: Use tools and linters to catch mistakes.

### Setting Up ESLint for Hooks

Install the React Hooks ESLint plugin to enforce best practices:

```jsx
npm install -D eslint-plugin-react-hooks@4.6.0
```

Update your `.eslintrc.json` file:

```jsx
{
    "extends": [
        "plugin:react-hooks/recommended"
    ]
}
```

Ensure that this plugin is listed before Prettier in the `extends` array.

### Benefits of the Plugin

The ESLint plugin for React hooks:

- Warns against incorrect hook usage (e.g., calling hooks conditionally or inside loops).
- Helps catch common pitfalls, making your codebase more reliable and maintainable.

## Running ESLint

To see the ESLint plugin in action:

```sh
npm run dev
```

If there are issues like hooks inside conditionals, ESLint will warn you. Although these warnings can be mildly annoying, they are valuable for maintaining high-quality code.
