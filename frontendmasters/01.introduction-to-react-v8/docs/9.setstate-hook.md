# SearchParams Component: Managing Form Inputs with React Hooks

This section demonstrates creating a `SearchParams` component for searching pets by location. It covers **React Hooks**, **controlled inputs**, and dynamic state management using `useState`.

## React Hooks and State Initialization

React state allows you to manage component-specific data dynamically. We use `useState` to create and manage the `location` value in the form.

```jsx
import React, { useState } from 'react';

const SearchParams = () => {
  const [location, setLocation] = useState('');

  return (
    <div className="search-params">
      <form>
        <label htmlFor="location">
          Location
          <input id="location" value={location} onChange={(e) => setLocation(e.target.value)} placeholder="Location" />
        </label>
        <button>Submit</button>
      </form>
    </div>
  );
};

export default SearchParams;
```

## Controlled Inputs

### Explanation

1. **State Binding**:

   - The `value` of the input is tied to the `location` state.

2. **Updating State**:

   - The `onChange` handler uses `setLocation` to update `location` as the user types.

3. **Controlled Behavior**:
   - React ensures the input reflects the current state (`location`) at all times.

## Controlled vs Uncontrolled Components

### Controlled

React state manages the component. Input changes trigger state updates, and React re-renders with the new state.

```jsx
<input value={location} onChange={(e) => setLocation(e.target.value)} />
```

### Uncontrolled

The DOM manages the component, requiring `ref` for accessing the input value manually.

```jsx
<input ref={inputRef} />
```

**React recommends controlled components for consistency and predictability.**

---

## Pitfalls and Best Practices

1. **Ensure Inputs Are Controlled**:
   Always use `value` and `onChange` for controlled components.

2. **Hooks Rules**:

   - Hooks like `useState` must be called at the **top-level** of the component.
   - Never place hooks inside loops or conditionals.

3. **Placeholder Initialization**:
   Default the input state with a placeholder or empty value:
   - `const [location, setLocation] = useState("");`

---

## Expected Behavior

1. **Initial State**:
   - Input starts empty or with a default placeholder.
2. **Dynamic Updates**:
   - Typing into the input updates the `location` state.
3. **Reactivity**:
   - React re-renders the input whenever the state changes.
