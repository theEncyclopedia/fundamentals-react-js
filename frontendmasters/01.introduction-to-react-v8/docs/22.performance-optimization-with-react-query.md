# Using React Query for Fetching and Displaying API Data in React

This section covers how to use React Query for API data fetching, caching, and error handling in a React application.

## Why React Query?

React Query simplifies API data management by:

- Automatically caching API responses.
- Providing robust error handling and retry mechanisms.
- Offering hooks like `useQuery` to simplify async data fetching logic.

## Step-by-Step Guide

### 1. Fetch Method Setup

In `FetchPet.js`, create a reusable method for fetching pet details:

```jsx
const fetchPet = async ({ queryKey }) => {
  const id = queryKey[1]; // Extract the pet ID from the query key
  const apiRes = await fetch(`http://pets-v2.dev-apis.com/pets?id=${id}`);

  if (!apiRes.ok) {
    throw new Error(`details/${id} fetch not ok`);
  }

  return apiRes.json(); // Return the parsed JSON data
};

export default fetchPet;
```

### Key Features

1. **Reusable Logic**: The fetch method is independent and testable.
2. **Error Handling**: Throws an error for unsuccessful responses (`apiRes.ok === false`).

### 2. Using `useQuery` in a Component

In `Details.jsx`, use `useQuery` to fetch pet data:

```jsx
import { useParams } from 'react-router-dom';
import { useQuery } from '@tanstack/react-query';
import fetchPet from './FetchPet';

function Details() {
  const { id } = useParams(); // Extract the pet ID from the route

  const { data, isLoading, isError } = useQuery(['pet', id], fetchPet);

  if (isLoading) {
    return (
      <div className="loading-pane">
        <h2 className="loader">🔄</h2>
      </div>
    );
  }

  if (isError) {
    return <h2>Error loading pet details</h2>;
  }

  const pet = data.pets[0]; // Extract the pet details

  return (
    <div className="details">
      <h1>{pet.name}</h1>
      <h2>{`${pet.animal} — ${pet.breed} — ${pet.city}, ${pet.state}`}</h2>
      <button>Adopt {pet.name}</button>
      <p>{pet.description}</p>
    </div>
  );
}

export default Details;
```

### How `useQuery` Works

1. **Query Key**:

   - Uniquely identifies the query (e.g., `["pet", id]`).
   - Used for caching and determining if data is already available.

2. **Fetch Method**:

   - Called only if data is not cached or has expired based on `staleTime` and `cacheTime`.

3. **States Provided by `useQuery`**:
   - **`isLoading`**: Indicates the query is in progress.
   - **`isError`**: Indicates the query failed.
   - **`data`**: Contains the API response.

### 3. Error Handling with React Query

React Query retries failed requests by default:

- Retries 3 times with exponential backoff.
- Provides `isError` and error states for custom handling.

#### Example

```jsx
if (isError) {
  return <h2>Error loading pet details</h2>;
}
```

### 4. Caching and Refetching

#### How Caching Works

- Data is cached using the query key (`["pet", id]`).
- Subsequent requests with the same key use the cached response unless explicitly invalidated.

#### Refetching Data

- Manually trigger a refetch using `refetch()` from the query result: `results.refetch()`.

## Benefits of React Query

1. **Simplified Logic**: Eliminates manual `useEffect` and `useState` for API requests.
2. **Optimized Performance**: Prevents redundant API requests with intelligent caching.
3. **Error and Retry Management**: Built-in retries for transient errors and easy error handling.
4. **Scalable Design**: Query keys allow granular control over data fetching and caching.

## Conclusion

React Query is a powerful tool for managing API data in React applications. By leveraging `useQuery`, developers can simplify async logic, improve caching, and ensure robust error handling, making React Query an essential addition to modern React workflows.
