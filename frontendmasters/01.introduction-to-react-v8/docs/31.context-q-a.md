# Advanced React Context Usage: Read and Write Access

This section details how to use React Context to enable both reading from and writing to global state across multiple components. The example showcases a pet adoption feature where adopting a pet updates a global context and displays the adopted pet's image in another component.

## Key Topics

### 1. Understanding Read and Write Access with Context

#### Reading from Context

- Use `useContext` to access the current value of the context.
- Example:

  ```jsx
  const adoptedPet = useContext(AdoptedPetContext)[0];
  ```

#### Writing to Context

- Pass a state-updating function (e.g., `setState`) via context to allow updates.
- Example:

  ```jsx
  const setAdoptedPet = useContext(AdoptedPetContext)[1];
  ```

- **Combined Example:**

  - Pass both the state and the updater function through context:

    ```jsx
    <AdoptedPetContext.Provider value={[adoptedPet, setAdoptedPet]}>{children}</AdoptedPetContext.Provider>
    ```

### 2. Writing to Context: Example in `Details.jsx`

#### Code Example

- **Using Context and Navigation:**

  ```jsx
  import React, { useContext } from 'react';
  import { useNavigate } from 'react-router-dom';
  import AdoptedPetContext from './AdoptedPetContext';

  function Details({ pet }) {
    const navigate = useNavigate();
    const [, setAdoptedPet] = useContext(AdoptedPetContext);

    return (
      <div>
        <h1>Would you like to adopt {pet.name}?</h1>
        <button
          onClick={() => {
            setAdoptedPet(pet);
            navigate('/');
          }}
        >
          Yes
        </button>
      </div>
    );
  }

  export default Details;
  ```

#### Key Points

- The second element of the array from `useContext` is used to update the context state (`setAdoptedPet`).
- Navigation (`navigate("/")`) redirects the user to the homepage upon adoption.

### 3. Reading from Context: Example in `SearchParams.jsx`

#### Code Example

- **Accessing and Displaying Context:**

  ```jsx
  import React, { useContext } from 'react';
  import AdoptedPetContext from './AdoptedPetContext';

  function SearchParams() {
    const [adoptedPet] = useContext(AdoptedPetContext);

    return (
      <div>
        {adoptedPet && (
          <div className="pet image-container">
            <img src={adoptedPet.images[0]} alt={adoptedPet.name} />
          </div>
        )}
      </div>
    );
  }

  export default SearchParams;
  ```

#### Key Points

- Only the state (first element of the array) is accessed as the component doesn't need to update it.
- Conditional rendering ensures the adopted pet's image is only displayed when a pet is adopted.

### 4. Best Practices and Considerations

#### 4.1 Best Practices

1. **Minimal Use:** Use context sparingly for truly global state.
2. **Clear Intentions:** Name variables explicitly (`adoptedPet` vs. `_`) to clarify usage.
3. **Encapsulation:** Wrap context logic in reusable hooks for better abstraction (e.g., `useAdoptedPet`).

#### 4.2 Trade-offs

1. **Indirection:** Context introduces implicit data flow, making debugging harder.
2. **Performance:** Frequent updates to large contexts can trigger unnecessary re-renders.

### 5. Summary of Workflow

1. **Define Context:** Create a context to hold global state and updater functions.
2. **Provide Context:** Use `Context.Provider` to wrap components requiring access to the context.
3. **Consume Context:** Use `useContext` to access or update context values.

## Conclusion

React Context provides a powerful way to share state globally across your application. By balancing judicious use of context with explicit state management, you can achieve maintainable and scalable applications. Always evaluate your application's needs to decide whether context, prop drilling, or a more robust state management library like Redux is appropriate.
