# React Query and Uncontrolled Forms Refactor

This section details the process of refactoring a React component to replace `useEffect` with `React Query` for data fetching and introduces the concept of uncontrolled forms. The focus is on leveraging `useQuery` for robust API interaction, achieving better caching, and simplifying state management.

## Key Topics

### 1. Removing `useEffect`

- **Why remove `useEffect`:**

  - Simplifies the codebase.
  - Avoids excessive rerenders due to dependency tracking.
  - Enables better caching and error handling.

- **Replacement Strategy:**

  - Use `useQuery` from `@tanstack/react-query` for fetching data.
  - Define query keys to manage caching.

### 2. Creating Uncontrolled Forms

- **Definition:**

  - Uncontrolled forms rely on the DOM to manage form state rather than React's `useState`.

- **Advantages:**

  - Reduces the amount of state managed in React.
  - Browser APIs handle form state and data extraction.

- **Implementation Steps:**
  1. Remove `value` and `onChange` handlers for form fields.
  2. Add `name` attributes to form inputs.
  3. Use `FormData` API to extract form data on form submission.

### 3. Setting Up React Query

- **Data Fetching:**

  - Moved logic into a reusable function `fetchSearch`.
  - `fetchSearch` performs asynchronous API calls and handles errors.

- **Usage in Component:**

  - Define `requestParams` state to store query parameters.
  - Pass `requestParams` and `fetchSearch` to `useQuery`.

- **Advantages of `useQuery`:**
  - Built-in caching.
  - Automatic refetching on dependency changes.
  - Simplified error and loading state handling.

## Code Examples

### Fetch Function

```jsx
export default async function fetchSearch({ queryKey }) {
  const [_, { animal, location, breed }] = queryKey;
  const res = await fetch(`http://pets-v2-dev-apis.com/pets?animal=${animal}&location=${location}&breed=${breed}`);

  if (!res.ok) {
    throw new Error('Pet search not okay');
  }

  return res.json();
}
```

### Refactored Form Submission

```jsx
const handleSubmit = (e) => {
  e.preventDefault();

  const formData = new FormData(e.target);
  const obj = {
    animal: formData.get('animal') || '',
    location: formData.get('location') || '',
    breed: formData.get('breed') || '',
  };

  setRequestParams(obj);
};
```

### Using `useQuery` in the Component

```jsx
const { data, isLoading, error } = useQuery(['search', requestParams], fetchSearch, { enabled: !!requestParams });
const pets = data?.pets || [];
```

## Lessons Learned

1. **Uncontrolled Forms:**

   - Simplify input handling by offloading state management to the browser.

2. **React Query Benefits:**

   - Avoids manual state management and error handling.
   - Streamlines the codebase, especially for API calls.

3. **Code Organization:**
   - Keep fetch logic in separate files for reusability.
   - Consider modular folder structures for better maintainability.

## Summary

By replacing `useEffect` with `useQuery` and adopting uncontrolled forms, the codebase becomes:

1. Easier to read.
2. More performant with built-in caching.
3. Robust with enhanced error handling.

This refactor demonstrates how to leverage modern React tools effectively for cleaner and more maintainable code.
