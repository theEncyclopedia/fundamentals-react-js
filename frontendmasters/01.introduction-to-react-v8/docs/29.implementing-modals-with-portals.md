# Integrating Modal with Portals in React

This section demonstrates how to use the `Modal` component built with React portals. It explains integrating it into a page, triggering it with state, handling button clicks inside the modal, and using parent state to manage modal actions.

## Key Topics

### 1. Adding the Modal to a Component

#### Step 1: Import and Use `Modal`

- Import `Modal` and `useState` to manage modal visibility.
- Example:

  ```jsx
  import React, { useState } from 'react';
  import Modal from './Modal';

  function Details() {
    const [showModal, setShowModal] = useState(false);

    return (
      <div>
        <button onClick={() => setShowModal(true)}>Adopt Pet</button>
        {showModal && (
          <Modal>
            <div>
              <h1>Would you like to adopt this pet?</h1>
              <div className="buttons">
                <button onClick={() => alert('Yes clicked')}>Yes</button>
                <button onClick={() => setShowModal(false)}>No</button>
              </div>
            </div>
          </Modal>
        )}
      </div>
    );
  }

  export default Details;
  ```

#### Step 2: Conditional Rendering for the Modal

- Use a state variable (`showModal`) to toggle modal visibility.
- Render the `Modal` component conditionally based on this state.

### 2. Using React Portal for Rendering

- The modal content is rendered in a separate DOM node using a portal.
- Despite rendering outside the main DOM hierarchy, the modal has full access to the parent component’s state and props.

### 3. Handling Button Clicks in the Modal

#### Method 1: Directly Handle Clicks

- Assign unique handlers to each button.
- Example:

  ```jsx
  <button onClick="{()" ="">alert("Yes clicked")}>Yes</button>
  <button onClick="{()" ="">setShowModal(false)}>No</button>
  ```

#### Method 2: Pass Event Info to Parent

- Use the `onClick` event's parameter to identify the clicked button.
- Example:

  ```jsx
  function handleButtonClick(e) {
      const buttonValue = e.target.innerText;
      console.log(buttonValue); // Logs "Yes" or "No"
  }

  <button onClick={handleButtonClick}>Yes</button>
  <button onClick={handleButtonClick}>No</button>
  ```

#### Method 3: Use Dynamic Event Handlers

- Assign specific functions to `onClick` for each button.
- Example:

  ```jsx
  <button onClick={() => performAction("yes")}>Yes</button>
  <button onClick={() => performAction("no")}>No</button>
  ```

### 4. Benefits of Portals in Modals

- **CSS Layering:**
  - Modals render above other elements without interfering with the component tree structure.
- **Access to Parent State:**
  - Full access to the state and props of the parent component.
- **Event Handling:**
  - Events inside the modal bubble up through React's virtual DOM, allowing flexible event handling.

## Lessons Learned

1. **State Management:**
   - Use `useState` to manage modal visibility and handle actions within the modal.
2. **Portals:**
   - Render modals outside the parent DOM hierarchy for clean and maintainable UI.
3. **Event Handling:**
   - Assign specific handlers for better clarity and control over modal button actions.

## Summary

By combining React portals and modals, you can create user-friendly overlays that seamlessly interact with parent state and logic. This approach simplifies UI development, reduces state management complexity, and ensures maintainable and reusable code.
