# Accessing Route Parameters with `useParams` in React Router

This document explains how to extract route parameters using the `useParams` hook in React Router, along with an overview of React Router context and a practical example.

---

## What is `useParams`?

The `useParams` hook allows you to access dynamic parameters from the current route. These parameters are defined in the route's path, such as `/details/:id`, where `:id` is a dynamic parameter.

---

## Using `useParams` in a Component

### Example: Accessing the `id` Parameter

In `Details.jsx`:

```jsx
import { useParams, Link } from 'react-router-dom';

function Details() {
  // Extract the "id" parameter from the route
  const { id } = useParams();

  return (
    <div>
      <h1>Pet Details</h1>
      <p>Pet ID: {id}</p>

      {/* Add a home link */}
      <header>
        <Link to="/">Adopt Me!</Link>
      </header>
    </div>
  );
}

export default Details;
```

### Key Concepts

1. **Dynamic Parameters**:

   - `useParams` retrieves values from dynamic segments of the route, such as `:id` in `/details/:id`.

2. **Context Dependency**:
   - `useParams` relies on a `BrowserRouter` or equivalent router component wrapping the app. Without it, `useParams` will throw an error.

---

## How Does `useParams` Work?

React Router uses context to provide routing information to components:

1. The `BrowserRouter` wraps the application in `App.jsx`.
2. The `Route` component defines the path and dynamic parameters.
3. `useParams` hooks into the context provided by `BrowserRouter` to extract route parameters.

### Example: Route Definition in `App.jsx`

```jsx
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import SearchParams from './SearchParams';
import Details from './Details';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<SearchParams />} />
        <Route path="/details/:id" element={<Details />} />
      </Routes>
    </BrowserRouter>
  );
}

export default App;
```

---

## Adding a Home Link

A common feature in applications is a navigation link to return to the home page. Use the `Link` component to create a client-side navigation link:

```jsx
<header>
  <Link to="/">Adopt Me!</Link>
</header>
```

### Benefits of `Link` Over `<a>`

- Prevents full-page reloads by handling navigation on the client side.
- Maintains app state and improves performance.

2. **Using DevTools**:

   - Inspect React Dev Tools to verify context and params.
   - Look at the `props` and `context` of your component in the React Dev Tools hierarchy.

3. **Dynamic Parameters**:
   - Check that your route path includes the correct dynamic parameter (e.g., `:id`).
   - Ensure the component is rendered by a matching `Route`.

The `useParams` hook is a powerful tool for accessing dynamic route parameters in React Router.
