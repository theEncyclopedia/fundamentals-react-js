# Using `useEffect()` for API Request in React

This section covers the use of React's `useEffect` hook to make API requests for dynamically updating data in response to user interactions. It also addresses handling dependencies, preventing re-renders, and best practices for working with external APIs.

## Introduction to `useEffect`

The `useEffect` hook is used to handle side effects in React components. Common use cases include:

- Fetching data from APIs.
- Interacting with local storage.
- Managing operations outside the component lifecycle.

In this context, `useEffect` is employed to fetch and display data (e.g., pets) from an API based on user input.

## Setting Up the Component

Here’s the component setup to manage location, animal, breed, and fetch data dynamically:

```jsx
import { useState, useEffect } from 'react';

function SearchParams() {
  const [location, setLocation] = useState('');
  const [animal, setAnimal] = useState('');
  const [breed, setBreed] = useState('');
  const [breeds, setBreeds] = useState([]);
  const [pets, setPets] = useState([]);

  // Fetch data on component load
  useEffect(() => {
    requestPets();
  }, []); // Empty array ensures this runs only once

  async function requestPets() {
    const res = await fetch(
      `https://pets-v2.devdashapis.com/pets?animal=${animal}&location=${location}&breed=${breed}`,
    );
    const json = await res.json();
    setPets(json.pets);
  }

  const handleSubmit = (e) => {
    e.preventDefault();
    requestPets(); // Fetch data on form submit
  };

  return (
    <form onSubmit={handleSubmit}>
      <label htmlFor="location">
        Location
        <input id="location" value={location} onChange={(e) => setLocation(e.target.value)} />
      </label>

      <label htmlFor="animal">
        Animal
        <select
          id="animal"
          value={animal}
          onChange={(e) => {
            setAnimal(e.target.value);
            setBreed(''); // Clear breed on animal change
          }}
        >
          <option value="">Select an animal</option>
          <option value="bird">Bird</option>
          <option value="cat">Cat</option>
          <option value="dog">Dog</option>
        </select>
      </label>

      <label htmlFor="breed">
        Breed
        <select id="breed" value={breed} onChange={(e) => setBreed(e.target.value)} disabled={breeds.length === 0}>
          <option value="">Select a breed</option>
          {breeds.map((breed) => (
            <option key={breed} value={breed}>
              {breed}
            </option>
          ))}
        </select>
      </label>

      <button>Submit</button>

      <div>
        {pets.map((pet) => (
          <Pet key={pet.id} name={pet.name} animal={pet.animal} breed={pet.breed} />
        ))}
      </div>
    </form>
  );
}
```

## Key Concepts

### Fetching Data with `useEffect`

1. **Initial Data Fetch**:

   - To load data (e.g., a list of pets) when the component first renders, use an empty dependency array (`[]`).

   ```jsx
   useEffect(() => {
     requestPets();
   }, []); // Runs only on initial render
   ```

2. **Dynamic Updates**:

   - To refetch data whenever a specific variable changes (e.g., `animal`), include it in the dependency array:

   ```jsx
   useEffect(() => {
     requestPets();
   }, [animal]);
   ```

3. **Controlled Updates**:
   - Prevent continuous fetching by calling the function on specific events (e.g., form submission).

### Rendering Lists Dynamically

Use `.map()` to render data fetched from the API dynamically:

```jsx
<div>
  {pets.map((pet) => (
    <Pet key={pet.id} name={pet.name} animal={pet.animal} breed={pet.breed} />
  ))}
</div>
```

### Using Unique Keys

React requires a unique `key` for each item in the list to optimize re-renders. Use a unique identifier like `id` from the data source:

```jsx
pets.map((pet) => <Pet key={pet.id} {...pet} />);
```

Avoid using non-unique values like array indices.

### Handling Warnings and Dependencies

When `useEffect` dependencies are omitted or handled unconventionally:

1. Use a comment to explicitly disable the exhaustive-deps rule for specific lines.

   ```jsx
   // eslint-disable-next-line react-hooks/exhaustive-deps
   useEffect(() => {
     requestPets();
   }, []);
   ```

2. This approach is acceptable when you’re confident about your implementation and want to avoid unnecessary warnings.

---

By following these guidelines, you can efficiently manage API requests, dynamically update UI components, and optimize performance in your React applications.
