# React Error Boundaries

Error boundaries are a mechanism in React to catch JavaScript errors in components, preventing the entire app from crashing. They provide a way to gracefully handle errors, offering users an error message or recovery path. Error boundaries must be implemented using class components because they rely on lifecycle methods unavailable in functional components.

## Key Topics

### 1. What Are Error Boundaries?

- **Definition:**

  - React components that catch errors in their child component tree.
  - Display fallback UI instead of crashing the app.

- **Common Use Cases:**
  - Malformed API responses.
  - Unexpected user inputs causing crashes.
  - External service failures.

### 2. Lifecycle Methods for Error Boundaries

Error boundaries use two lifecycle methods exclusive to class components:

1. **`getDerivedStateFromError`:**

   - Updates the component state when an error is caught.
   - Example:

     ```js
     static getDerivedStateFromError(error) {
         return { hasError: true };
     }
     ```

2. **`componentDidCatch`:**

   - Logs error details for debugging or external tracking.
   - Example:

     ```jsx
     componentDidCatch(error, info) {
         console.error("ErrorBoundary caught an error", error, info);
         // Send error details to an external service, e.g., Sentry
     }
     ```

### 3. Creating an Error Boundary Component

- **Code Example:**

  ```jsx
  import React, { Component } from 'react';
  import { Link } from 'react-router-dom';

  class ErrorBoundary extends Component {
    state = { hasError: false };

    static getDerivedStateFromError(error) {
      return { hasError: true };
    }

    componentDidCatch(error, info) {
      console.error('ErrorBoundary caught an error', error, info);
    }

    render() {
      if (this.state.hasError) {
        return (
          <h2>
            There was an error with this listing. <Link to="/">Click here to go back to the homepage.</Link>
          </h2>
        );
      }

      return this.props.children;
    }
  }

  export default ErrorBoundary;
  ```

### 4. Using an Error Boundary

- **Wrapping Components:**

  - Wrap components prone to errors inside the `ErrorBoundary`.
  - Example:

    ```jsx
    import ErrorBoundary from './ErrorBoundary';

    function DetailsErrorBoundary() {
      return (
        <ErrorBoundary>
          <Details />
        </ErrorBoundary>
      );
    }

    export default DetailsErrorBoundary;
    ```

- **Why Wrap Specific Components:**
  - Error boundaries only catch errors in their child components.
  - Wrapping too high in the tree (e.g., entire app) may obscure specific error contexts.

### 5. Advanced Usage: Custom Error Messages

- **Customizing Error UI:**

  - Pass an `errorComponent` prop for custom fallback UI.
  - Example:

    ```jsx
    class ErrorBoundary extends Component {
      render() {
        if (this.state.hasError) {
          return this.props.errorComponent || <h2>Default error message</h2>;
        }

        return this.props.children;
      }
    }
    ```

- **Reusability:**
  - This approach allows the same error boundary to be used in various scenarios with custom error messages.

### 6. Best Practices

1. **Granularity:**

   - Use specific error boundaries for components prone to errors (e.g., user inputs, dynamic APIs).
   - Wrap higher in the tree for general-purpose error handling.

2. **Error Logging:**

   - Send errors to external services like Sentry or New Relic for tracking.

3. **Graceful Recovery:**
   - Provide users with actionable feedback (e.g., links to retry or navigate elsewhere).

## Lessons Learned

1. Error boundaries must use class components due to lifecycle methods like `getDerivedStateFromError` and `componentDidCatch`.
2. They enable graceful error handling and improve user experience by preventing app crashes.
3. Balancing the granularity of error boundaries depends on the app's complexity and error-prone areas.

## Summary

Error boundaries are essential for building resilient React applications. While they require class components, their ability to catch and handle errors enhances app stability and user experience. Proper implementation includes using granular boundaries, logging errors for debugging, and providing intuitive fallback UI.
