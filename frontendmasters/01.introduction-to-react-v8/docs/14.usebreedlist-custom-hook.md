# Building a Custom Hook: `useBreedList` in React

This section demonstrates how to create a custom React hook, `useBreedList`, to fetch and cache breed data based on the selected animal. It also covers best practices for handling asynchronous operations within React's `useEffect`.

## Introduction

Custom hooks in React package reusable logic involving other hooks like `useState` and `useEffect`. The `useBreedList` hook encapsulates functionality to fetch a list of breeds dynamically based on the selected animal, leveraging caching to avoid redundant API requests.

## Implementing `useBreedList`

### Hook Definition

Create a new file, `useBreedList.js`, and define the custom hook:

```jsx
import { useState, useEffect } from 'react';

const localCache = {};

export default function useBreedList(animal) {
  const [breedList, setBreedList] = useState([]);
  const [status, setStatus] = useState('unloaded');

  useEffect(() => {
    if (!animal) {
      setBreedList([]);
    } else if (localCache[animal]) {
      setBreedList(localCache[animal]);
    } else {
      requestBreedList();
    }

    async function requestBreedList() {
      setBreedList([]);
      setStatus('loading');

      const res = await fetch(`https://pets-v2.dev-apis.com/breeds?animal=${animal}`);
      const json = await res.json();

      localCache[animal] = json.breeds || [];
      setBreedList(localCache[animal]);
      setStatus('loaded');
    }
  }, [animal]);

  return [breedList, status];
}
```

### Key Features

1. **Local Caching**:

   - Prevents redundant API calls by storing fetched breeds in a local object (`localCache`).
   - If the user switches between animals, previously fetched data is served from the cache.

2. **State Management**:

   - `breedList`: Holds the current list of breeds.
   - `status`: Tracks the loading state (`unloaded`, `loading`, `loaded`) for better user experience and testing.

3. **Dependency Management**:

   - The `useEffect` re-runs only when the `animal` value changes.

4. **Async Function Inside `useEffect`**:
   - To handle `async/await` within `useEffect`, define the async function inside the effect and call it.

## Integrating `useBreedList` in `SearchParams`

In `SearchParams`, use the custom hook to dynamically fetch and render breed options:

```jsx
import { useState } from 'react';
import useBreedList from './useBreedList';

function SearchParams() {
  const [animal, setAnimal] = useState('');
  const [breed, setBreed] = useState('');
  const [breeds] = useBreedList(animal);

  return (
    <form>
      <label htmlFor="animal">
        Animal
        <select
          id="animal"
          value={animal}
          onChange={(e) => {
            setAnimal(e.target.value);
            setBreed(''); // Reset breed when animal changes
          }}
        >
          <option value="">Select an animal</option>
          <option value="bird">Bird</option>
          <option value="cat">Cat</option>
          <option value="dog">Dog</option>
        </select>
      </label>

      <label htmlFor="breed">
        Breed
        <select id="breed" value={breed} onChange={(e) => setBreed(e.target.value)} disabled={breeds.length === 0}>
          <option value="">Select a breed</option>
          {breeds.map((b) => (
            <option key={b} value={b}>
              {b}
            </option>
          ))}
        </select>
      </label>
    </form>
  );
}
```

## Benefits of Using a Custom Hook

### Encapsulation

Encapsulates complex logic (e.g., caching and API requests) into a reusable function, keeping components clean and focused.

### Reusability

The `useBreedList` hook can be reused across components or projects without modification.

### Testability

Tracking the `status` state simplifies testing. For example, tests can verify that the hook transitions from "unloaded" to "loading" and finally to "loaded."

## Best Practices for Async Operations in `useEffect`

1. **Async Inside `useEffect`**:

   - `useEffect` itself cannot be `async`. To use `async/await`, define and invoke an `async` function inside `useEffect`.

   ```jsx
   useEffect(() => {
     async function fetchData() {
       // Perform async operations here
     }
     fetchData();
   }, [dependencies]);
   ```

2. **Dependency Management**:

   - Include all variables used inside the `useEffect` in its dependency array to avoid stale closures.
   - Suppress ESLint warnings judiciously if the behavior diverges from recommendations (e.g., intentional exclusion of dependencies).

3. **Caching**:
   - Cache fetched data to reduce redundant requests and improve performance.

3. Switch back to "dog" to verify no additional request is made, as the data is served from `localCache`.
