# Component Composition in React: Creating Reusable Components

This document covers best practices for component composition in React, specifically extracting reusable components to simplify your codebase and improve maintainability.

---

## When to Extract a Component

Consider extracting parts of your code into reusable components when:

- The code can be reused in different parts of the application (e.g., displaying a list of pets).
- A component becomes too large to manage (e.g., exceeding ~100 lines of code).
- You want to isolate and test specific parts of your application independently.

## Example: Creating a `Results` Component

We’ll extract the logic for rendering a list of pets into a reusable `Results` component.

### Implementation

1. **Create the `Results.jsx` File**

Create a new file in the `src` directory and name it `Results.jsx`.

```jsx
import Pet from './Pet';

const Results = ({ pets }) => {
  return (
    <div className="search">
      {pets.length === 0 ? (
        <h1>No Pets Found</h1>
      ) : (
        pets.map((pet) => (
          <Pet
            key={pet.id}
            name={pet.name}
            animal={pet.animal}
            breed={pet.breed}
            images={pet.images}
            location={`${pet.city}, ${pet.state}`}
          />
        ))
      )}
    </div>
  );
};

export default Results;
```

### Key Concepts

1. **Destructuring Props**:

   - The `Results` component expects a `pets` prop, destructured from the incoming props.
   - This improves readability and ensures that only the required data is used.

2. **Conditional Rendering**:

   - If `pets` is an empty array, display a message (`No Pets Found`).
   - Otherwise, map through the array and render each pet using the `Pet` component.

3. **Explicit Props**:

   - Each property (`name`, `animal`, `breed`, etc.) is passed explicitly to the `Pet` component.
   - Avoid using the spread operator (`...pet`) unless:
     - The component is a pure pass-through.
     - The structure of the data is guaranteed to remain stable.

4. **Adding a Unique `key`**:
   - The `key` prop ensures React can efficiently update the DOM when the list changes.
   - Always use a unique identifier like `id` for the `key`.

## Refactoring `SearchParams`

After creating the `Results` component, update `SearchParams` to use it.

```jsx
import { useState, useEffect } from 'react';
import Results from './Results';

function SearchParams() {
  const [pets, setPets] = useState([]);

  useEffect(() => {
    async function fetchPets() {
      const res = await fetch(`https://pets-v2.dev-apis.com/pets?animal=dog`);
      const json = await res.json();
      setPets(json.pets);
    }
    fetchPets();
  }, []);

  return (
    <div>
      <form>{/* Form inputs here */}</form>
      <Results pets={pets} />
    </div>
  );
}

export default SearchParams;
```

## Why Smaller Components Are Better

1. **Reusability**:

   - Components like `Results` can be used across multiple parts of the application (e.g., an admin page).

2. **Readability**:

   - Smaller components are easier to understand and maintain.

3. **Testability**:
   - Isolated components can be tested independently, improving the robustness of your codebase.

---

## Using the Spread Operator: A Word of Caution

While the spread operator (`...`) can simplify passing props, it has drawbacks:

- **Implicit Behavior**:
  - Passing all properties might include unintended data.
  - Future changes to the data structure can break the component.
- **Readability**:
  - Explicitly listing props improves clarity and ensures you’re only passing what’s needed.

### Appropriate Use Case

Use the spread operator in pass-through components where all props are forwarded without modification.

```jsx
const Wrapper = (props) => <SomeComponent {...props} />;
```

1. Displays a list of pets dynamically.
2. Shows a "No Pets Found" message when the array is empty.
3. Adds styling via the `search` class on the parent `div`.

When searching for pets:

Extracting reusable components like `Results` simplifies development, improves maintainability, and aligns with React’s compositional design philosophy.
